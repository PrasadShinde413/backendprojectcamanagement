## 🎯 How It Works

```
STEP 1: Create WorkService with is_recurring=True
        ↓
STEP 2: Use /works/bulk-create/ API with is_recurring=True
        ↓
STEP 3: RecurringWorkAssignment auto-created & employees stored
        ↓
STEP 4: Run management command monthly
        ↓
STEP 5: System creates new Work for next month with same employees
        ↓
STEP 6: Process repeats automatically each month
```

## 🚀 Usage Examples

### Example 1: Create Recurring Monthly Audit

```bash
# 1. Create WorkService
POST /api/work-services/create/
{
    "service_name": "Monthly Compliance Audit",
    "description": "Monthly compliance audit for all clients",
    "is_recurring": true
}

# 2. Assign to employees
POST /api/works/bulk-create/
[{
    "assignment": 1,
    "work_service": 5,
    "price": "50000.00",
    "advance_payment": "10000.00",
    "work_mode": "Fixed",
    "assigned_employees": [1, 2, 3],
    "is_recurring": true
}]

# 3. Auto-generate for next month
python manage.py create_recurring_works --month 2024-03-01
```

### Example 2: Schedule Monthly Execution

```bash
# Add to crontab
0 0 1 * * cd /path/to/project && python manage.py create_recurring_works
```

## 🔐 Data Integrity

### Duplicate Prevention
```python
# Check if work exists before creating
existing = Work.objects.filter(
    assignment=assignment,
    work_service=work_service,
    created_for_month=target_month,
    is_deleted=False
).exists()

if existing:
    skip_creation()  # ← Prevents duplicates
else:
    create_work()
```

### Employee Validation
```python
# All employees must exist and be active
valid_ids = User.objects.filter(
    id__in=employee_ids,
    is_active=True
).values_list('id', flat=True)

missing = set(employee_ids) - set(valid_ids)
if missing:
    raise ValidationError(f"Invalid employees: {missing}")
```
